# Proposal: Use Forecaster Models

## Problem
Currently, the live demo uses "Perfect Foresight" where the predicted demand exactly matches the actual load generated by the simulation engine. This oversimplifies the agent's task natively avoiding the realism of forecasting errors and uncertainties. Furthermore, there is no automated pipeline to seamlessly train, compare, and select the best forecasting model for the simulation to use.

## Proposed Solution
We will implement an automated model training and comparison script to select the best "Champion" model for each segment. Then, we will modify the `inference.py` script to use these trained machine learning forecaster models (e.g., LightGBM) to provide a realistic, noisy "Predicted Demand" to the simulated environment rather than the perfectly accurate ground truth. The environment's battery logic will then act on this imperfect prediction instead of the actual simulated demand.

## Capabilities

### New Capabilities
- `model-selection`: A script (`train_and_compare_models.py`) to systematically train forecaster models, compare them via walk-forward validation against baseline champion models, and automatically promote the best performer.
- `realistic-inference`: Modified `inference.py` to use `.joblib` model predictions for the forecasted demand, creating realistic forecasting errors.

### Modified Capabilities
- `energy-trading-env`: Continues to receive `forecasted_demand` from `inference.py` but will now be dealing with imperfect forecasts, forcing the RL agent to adapt to realistic market uncertainties.

## Impact
- **Simulation Env**: The `EnergyTradingEnv` will face more realistic challenges and might require retraining of the PPO agent to handle noisy predictions.
- **Automation Scripts**: New scripts in `scripts/` will finalize the model selection process previously sketched out in notebooks.
